## Дизайн-система

| Параметр | Значение |
|----------|----------|
| Шрифт | Golos Text (Google Fonts), wght 400–900 |
| Акцент | `#5B4CF5` (фиолетовый) + gradient `#5B4CF5 → #7C3AED` |
| Фон страницы | `#F4F5F7` |
| Карточки | `#FFFFFF` с цветным левым бордером 3px |
| Сайдбар | Белый, активный пункт — градиент с тенью |
| Тени | Многоуровневые: `--sh`, `--sh-md`, `--sh-lg` |
| Радиус | `--r: 12px`, `--rs: 8px` |

CSS-переменные описаны в `:root` в начале `<style>`.

## Cloudinary

Все изображения и файлы хранятся в Cloudinary (не в Firestore и не в Firebase Storage).

| Параметр | Значение |
|----------|----------|
| Cloud name | `dywtdq1yo` |
| Upload preset | `Design` (unsigned) |
| Endpoint | `https://api.cloudinary.com/v1_1/dywtdq1yo/image/upload` |

Используется для:
- Изображений в блоках презентации
- Изображений в блоках лендинга
- Изображений в блоках наружной рекламы
- Макетов с пометками (outdoor edit mockup)
- Файлов результата работы в модалке задачи
- Файлов референса и доп. материалов (полиграфия, кампания)

Функция-хелпер: `uploadToCloudinary(file)` — async, возвращает `secure_url`.

## Роли

### Постановщик задачи
- Создаёт задачи через форму ТЗ
- Видит загруженность дизайнеров (вкладка "Загрузка дизайнеров")
- Видит статусы своих задач
- Комментирует задачи
- Редактирует задачи после отправки
- **НЕ двигает карточки по доске**
- **НЕ меняет статус задачи**

### Дизайнер
- Видит канбан-доску с 4 колонками
- Двигает карточки между колонками (drag-and-drop)
- Меняет статус задачи
- Редактирует все поля задачи
- Комментирует задачи
- Видит стикеры изменений на карточках
- Выбирает динозавра-аватар при настройке профиля (9 вариантов)

---

## Авторизация
**Постановщик:** вводит имя (свободный ввод)
**Дизайнер:** выбирает из списка в Firebase → вводит имя → выбирает динозавра и цвет

Данные профиля (имя, цвет, аватар) сохраняются в Firebase.
Цвет дизайнера используется для левого бордера его карточек и аватара в списке.

## Канбан дизайнера

### 4 колонки с цветными полосами сверху:
| Колонка | Цвет полосы |
|---------|-------------|
| Новое | `#4285f4` синий |
| В работе | `#fbbc04` жёлтый |
| На правках | `#ea4335` красный |
| Готово | `#34a853` зелёный |

### Карточка на доске:
- Строка 1: тип задачи (эмодзи 16px + название 10px uppercase) + приоритет-бейдж
- Строка 2: название задачи (15px, bold, до 2 строк)
- Разделитель
- Строка 3: аватар + имя дизайнера + дедлайн
- Фон карточки: чистый белый `#FFFFFF`
- Левый бордер 3px = цвет дизайнера
- Просроченный дедлайн — красный цвет
- Hover: подъём `translateY(-2px)` + усиленная тень

### Приоритет-бейджи:
- Срочный → `#FEE2E2 / #DC2626`
- Высокий → `#FEF3C7 / #D97706`
- Обычный → `#F0FDF4 / #16A34A`

### Стикеры изменений (правый верхний угол карточки):
- 💬 + число = новые комментарии
- ✏️ + число = изменения полей
- 🔄 + число = смена статуса
- Пропадают при открытии карточки, появляются снова при новом изменении

### Модалка по клику на карточку:
**Шапка:** тип + приоритет + название + статус (меняет только дизайнер)
**Основной блок:** исполнитель / дедлайн / описание / ТЗ от постановщика (все поля)
**Результат работы:** ссылка + загрузка файла (только дизайнер)
**Лог изменений:** автоматические записи изменений полей
**Комментарии:** лента с именем + датой, поле ввода (пишут оба)

### ТЗ в модалке (buildTZBlock):
Показывает все поля для каждого типа задачи. Полностью проаудировано и исправлено — все сохранённые поля отображаются корректно.

### Права редактирования:
| Поле | Постановщик | Дизайнер |
|------|:-----------:|:--------:|
| Название | ✅ | ✅ |
| Описание | ✅ | ✅ |
| Дедлайн | ✅ | ✅ |
| Приоритет | ✅ | ✅ |
| Статус | ❌ | ✅ |
| Результат работы | ❌ | ✅ |
| Комментарии | ✅ | ✅ |

## Типы задач и формы ТЗ

### 🎬 Видео
- Платформа (pill-теги): YouTube / TikTok / Instagram / ВКонтакте / Rutube / Другое
- Ориентация: Горизонтальная 16:9 / Вертикальная 9:16 / Квадрат 1:1
- Разрешение (текстовое поле, например 1920x1080)
- Длительность (сек)
- Формат файла (pill-теги): MP4 / MOV / AVI / GIF / Другой

### 📱 Соцсети
- Платформа + форматы (Сторис / Пост / Вертикальный пост / GIF)
- 9 карточек контента в сетке (сворачиваемые)
- Каждая карточка: текстовое поле + загрузка файла

### 📊 Презентация
**3 сценария:** Создать новую / Внести правки / Адаптировать под новый объект

- Редактор слайдов (16:9), блоки Текст и Изображение
- Текстовые теги: Заголовок / Основной текст / Список / Цифра/факт / Подпись
- Изображения: загрузка через Cloudinary + пометки-стикеры + полноэкранный режим
- Статус слайда: Готово к работе / Нужно обсудить
- Ссылка на файл / Заметки адаптации / Референс / Доп. материалы

### 🌐 Лендинг
**3 сценария:** Создать новый / Внести правки / Адаптировать

- Блочный редактор секций (типы: Главный экран / Описание / Преимущества / и др.)
- Внутри каждой секции: блоки Текст и Изображение (Cloudinary)
- Ссылка на файл / Заметки адаптации / Референс / Доп. материалы

### 🏙️ Наружная реклама
**3 сценария:** Новый макет / Внести правки / Адаптация под форматы

- Тип носителя, размер (мм + dpi), анимация
- Блочный редактор контента с тегами
- Макет с пометками (Cloudinary) для правок
- Адаптация: загрузка оригинала + выбор форматов + размеры

### 🖨️ Полиграфия
**4 категории:** Презентационный формат / Офисная полиграфия / Сувенирка / Навигация

- Типы, форматы, тираж, двусторонняя печать
- Референс стиля (Cloudinary) + Дополнительные материалы (Cloudinary)

### ✨ Фирменный стиль
- Элементы (чекбоксы): Логотип / Цвета / Типографика / Паттерн / Визитки / Бланки / Соцсети / Гайдлайн

### ✏️ Правки
- Ссылка на файл + описание правок

### 📣 Рекл. кампания
- Объект (список объектов)
- Цель кампании
- Носители (чекбоксы): Билборд / Баннер на забор / Сити-лайт / Digital-экран
- Референс стиля (Cloudinary) + Дополнительные материалы (Cloudinary)

## Общие технические детали

### Firebase
- Все задачи хранятся в Firestore
- Realtime обновления — все видят изменения без перезагрузки
- Test mode (открытый доступ, периодически нужно обновлять правила)
- Список дизайнеров тянется из Firebase (не хардкод)

### Пометки на изображениях
- Жёлтые стикеры по клику на изображение
- Перетаскиваемые, с автопереносом текста
- Полноэкранный режим с теми же пометками
- Синхронизация пометок между превью и полным экраном

### Вставка изображений
- Поддержка Ctrl+V в блоках изображений (pres, landing, outdoor)
